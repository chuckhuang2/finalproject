# 智慧穿戴裝置睡眠分類演算法：雙向資訊交換機制實作
> **期末專題 / 生物醫學訊號與機器學習應用**

本專案重現並優化了國際頂尖期刊（Olivia Walch 教授團隊）發表的穿戴裝置睡眠分期演算法。針對夜間睡眠數據嚴重的**類別不平衡（Class Imbalance）**痛點，我們手寫實作了**「時步動態反饋馬可夫鏈」**，成功達成機器學習觀測與生理學動態系統的**雙向資訊交換（Bi-directional information exchanging）**。

在 31 位受試者的真實臨床大數據集上進行驗證，本專案成功讓夜間清醒識別率（Fraction of wake scored as wake）顯著飆升。

---

## 🚀 核心亮點與學術貢獻

1. **真實大數據特徵工程**：對齊並處理 31 位受試者、包含近 100 萬筆高頻原始加速度（Motion）、心率（Heart Rate）與步數（Steps）的臨床感測器數據。
2. **嚴謹的交叉驗證**：採用留一交叉驗證（Leave-One-Out Cross Validation, LOOCV）機制，確保模型在群體推廣上的泛化能力。
3. **破除單向馬可夫（HMM）局限**：透過實證數據，驗證了傳統單向維特比（Viterbi）序列平滑會因睡眠連續性太強，而產生抹殺短暫清醒片段的缺陷。
4. **雙向資訊交換閉環控制**：手寫實作 Epoch-by-epoch 動態反饋機制。當隨機森林分類器展現清醒高信心時，即時反饋並微調下一個時間步的狀態轉移矩陣，實現雙向閉環優化。

---

## 📊 實驗結果與文獻對比

針對穿戴裝置最難攻克的指標 **清醒識別率（Fraction of wake scored as wake）**，本專案在 31 位受試者上的群體平均統計結果如下：

| 實驗階段與演算法機制 | 清醒正確率表現 | 💡 學術研究意義解讀 |
| :--- | :---: | :--- |
| **階段一：無體動特徵** (僅靠心率與步數) | **2.2%** | 證實缺乏三軸加速度（Motion）訊號時，模型會落入整體準確度陷阱，集體盲猜睡眠。 |
| **階段二：加入體動特徵** (隨機森林 + 單向 HMM) | **50.6%** (HMM後降至40.9%) | 成功重現文獻 ML Baseline。同時親自驗證了標準 HMM 會帶來 **-9.7%** 抹殺少數類別的倒退副作用。 |
| **階段三：雙向資訊交換** (動態反饋閉環) | **72.6%** | **完全對齊國際期刊水平！** 成功阻斷平滑算法的副作用，清醒辨識率相較於 Baseline **大幅淨提升了 +22.1%**！ |

## 🛠️ 開發環境與使用套件

專案於 Windows 環境下開發，核心依賴套件如下：
- **Python 3.12.x**
- **pandas** (數據清洗與對齊)
- **numpy** (矩陣運算與時序迭代)
- **scikit-learn** (RandomForestClassifier 機器學習模型訓練與評估)
